# Repository Guidance

## Release Notes

When making major user-visible or platform changes to this site, update the Release notes section in `app/about/page.tsx` before finishing the task.

Add a new dated entry when the change is significant enough to mention publicly. For smaller related changes on the same day, append to the existing date entry.

Examples of changes that should update release notes:

- New data sources or major changes to Message Center, Roadmap, archive, RSS, SEO, or AI indexing behavior.
- New pages, navigation items, filters, or notable UI behavior.
- Performance improvements that affect how users browse the archive.
- Security or dependency upgrades that materially change the platform.

Keep entries concise, user-facing, and focused on what changed rather than implementation details.
